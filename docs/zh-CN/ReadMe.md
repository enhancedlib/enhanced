## 6.技术

### 关于这个项目中的泛型（模板）
类名后面跟着0的类（以下简称为“实现类”）通常是模板（泛型）类的通用实现类。

它们分离了模板（泛型）类的实现从模板按类型擦除（伪泛型）。

您不应该从另一个类继承实现类，也不应该直接实例化实现类。
正确的方法是实例化模板类，因为实现类没有公共方法。

另外，你不应该以实现类作为类型来实例化模板类，因为模板类通常是私有地继承实现类。
